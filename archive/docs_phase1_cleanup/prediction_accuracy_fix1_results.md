## B. Matching approach

1. For each display string, call **`player_registry.audit_player_identity_key`**:
   - Prefer hit in **`registry_metadata_lookup_map()`** (covers `registry_key`, `display_name`, `canonical_name`, `history_canonical_key`, `aliases`, and initials-style variants from **`_record_lookup_keys`**).
   - Else exact / **compact** index from **`_build_indexes(registry_players())`**.
   - Else **`_resolve_existing_registry_key`** (subset name fuzzy).
   - Else **`learner.normalize_player_key`** (`normalized_fallback`).

2. **Stable comparison key** for set operations: normalized **`registry_key`** when resolved, else normalized literal fallback.

3. **XI metrics:** `correct_picks = |act_keys ∩ pred_keys|`; missed/extra lists keep **scorecard / model display names** unchanged.

4. **`registry_bridged_overlap_count`:** for each key in the intersection, if some actual vs predicted pair has the same audit key but **different** naive `_nk` strings, count one “bridged” overlap (spelling / initials fixed by registry).

5. **Temporal flag:** `match_time.year` vs **`datetime.now(timezone.utc).year`** — informational only; never blocks the report.
